# DrugAware — เรียนรู้ • ป้องกัน • เข้าใจกฎหมาย

## Project Identity

DrugAware is an educational mobile application built with Flutter.

Its purpose is to provide knowledge and awareness through interactive exploration.

This application is NOT a game.

This application is NOT a mission-based activity.

This application is NOT intended to evaluate users.

The experience should feel calm, modern, visual, and educational.

Users should explore knowledge naturally.

---

# Primary Objective

Create an educational experience that helps students:

* Learn basic knowledge
* Understand impacts
* Explore prevention methods
* Understand legal principles
* Reflect through self-learning

The application should support school activities.

Target usage:
5–10 minutes.

---

# Core Philosophy

Design Principle:

Explore → Interact → Understand

NOT:

Complete → Score → Win

Users should never feel pressure.

No completion tracking.

No achievements.

No gamification.

No progress indicators.

---

# User Experience Goals

When users open the application:

1. They immediately understand what the app is.

2. They can explore content freely.

3. They discover interesting information.

4. They leave with useful knowledge.

The application should feel:

* Calm
* Friendly
* Modern
* Visual
* Lightweight

Avoid making it feel:

* Like homework
* Like exams
* Like a dashboard
* Like reading documents

---

# Product Direction

# Visual Direction

Style:

Material 3

Minimal

Soft

Readable

Offline-first

Colors:

Primary:
Deep Blue

Accent:
Emerald Green

Background:
Soft White

Use whitespace.

Avoid heavy gradients.

Avoid repeating card layouts.

---

# Interaction Rules

Do not overuse:

* Containers
* Card lists
* Accordions

Mix patterns:

* Hero
* Swipe
* Bottom Sheet
* Carousel
* Expand
* Animated transitions

Every screen should feel different.

---

# State Rules

Track ONLY:

* Dark Mode
* Reading Mode
* Font Scale
* Bookmarks

Never store:

* Progress
* Completion
* Scores
* Rewards
