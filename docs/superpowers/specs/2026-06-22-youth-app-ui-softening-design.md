# Youth Learning App UI Softening Design

## Goal

ปรับ UX/UI ของแอปทั้งระบบให้ดูนุ่มขึ้น อ่านง่ายขึ้น เป็นมิตรกับเยาวชนมากขึ้น และลดความรู้สึกเหมือนข้อสอบหรือการประเมิน โดยคงโครงสร้างหลักของแอป, navigation หลัก, หน้าเดิม, flow เดิม, และ feature หลักเดิมไว้

งานรอบนี้เป็นการ "เกลาประสบการณ์" ไม่ใช่การรื้อแอปใหม่

## Constraints

- ไม่เปลี่ยนโครงสร้างแท็บหลัก
- ไม่ย้ายหน้าหลักหรือเพิ่มหน้าใหม่
- ไม่ลบ feature หลัก เช่น Explore, Law, Quiz, Random Fact
- ไม่รีแฟกเตอร์เชิงสถาปัตยกรรมขนาดใหญ่
- แตะโค้ดให้น้อยที่สุด แต่ให้ผลลัพธ์ภาพรวมรู้สึกดีขึ้นจริง

## Product Direction

ประสบการณ์ที่ต้องการหลังปรับ:

- นุ่มและเป็นมิตรขึ้น
- ทันสมัยแต่ไม่ล้ำจนวุ่น
- อ่านง่ายขึ้น
- ลดความรู้สึกเหมือนข้อสอบ
- ลดความรู้สึกเหมือนการถูกตัดสินว่าถูกหรือผิด

## Scope

ไฟล์ที่จะปรับในรอบนี้:

- `lib/pages/home_tab.dart`
- `lib/pages/explore_tab.dart`
- `lib/pages/law_page.dart`
- `lib/pages/quiz_page.dart`
- `lib/pages/random_fact_page.dart`
- `lib/widgets/question_card.dart`
- `lib/constants/app_text.dart`

ไฟล์ที่ตั้งใจไม่แตะในรอบนี้ เว้นแต่จำเป็นมาก:

- `lib/widgets/main_navigation_screen.dart`
- `lib/state/app_state.dart`
- `lib/models/*`
- `lib/data/*`
- `lib/pages/topic_detail_page.dart`

## Design Changes

### 1. Home

เป้าหมายคือทำให้หน้าแรกโล่งขึ้นและมี hierarchy ชัดขึ้น โดยไม่เปลี่ยน layout หลัก

สิ่งที่จะทำ:

- เกลาข้อความ subtitle และข้อความรองให้ดูเป็นมิตรมากขึ้น
- เพิ่มคำอธิบายใต้ข้อความ "พลังของนักเรียนที่ช่วยเป็นเกราะป้องกันภัยยาเสพติด" เพื่อให้ความหมายชัดขึ้นและอบอุ่นขึ้น
- ลดความแข็งของ section title และ secondary text
- ทำให้ CTA ในแต่ละส่วนอ่านง่ายและรู้ว่ากดไปทำอะไร

ผลที่คาดหวัง:

- เด็กเปิดแอปแล้วเข้าใจเร็วขึ้นว่าแอปนี้ช่วยอะไร
- หน้าแรกดูชวนสำรวจมากกว่าดูเป็นศูนย์รวมการบ้าน

### 2. Explore

เป้าหมายคือคงจุดเด่นเรื่องการสำรวจและ swipe ไว้ แต่ทำให้สบายตาและเป็นมิตรมากขึ้น

สิ่งที่จะทำ:

- ปรับถ้อยคำของ empty state และ action labels ให้เบาลง
- ลดความแข็งของเงา, contrast และข้อความรองบางจุด
- ทำให้ action row ด้านล่างของ fact card ดูอ่านง่ายขึ้น

ผลที่คาดหวัง:

- ประสบการณ์ยังสนุกอยู่ แต่ไม่ดูรกหรือเร่งเร้าเกินไป

### 3. Law

เป้าหมายคือคงโครงสร้างเดิมและสถานการณ์เดิมไว้ แต่ soft ภาษาที่แรงหรือชี้ขาดเกินไป

สิ่งที่จะทำ:

- ปรับ wording ของผลลัพธ์ใน scenario ให้เป็นเชิงอธิบายมากขึ้น
- ลดคำที่ฟังดูขู่, ฟันธง, หรือกดดันเกินไป
- คง interaction เดิมไว้ทั้งหมด

ผลที่คาดหวัง:

- เนื้อหายังจริงจังได้ แต่เหมาะกับเยาวชนมากขึ้น

### 4. Quiz

เป้าหมายคือคง flow เดิมไว้ แต่ทำให้รู้สึกเป็นการทบทวนความเข้าใจมากกว่าการสอบ

สิ่งที่จะทำ:

- ลดหรือเบา progress cue ที่ชวนให้รู้สึกกดดัน
- ปรับข้อความหัวเรื่องและข้อความรองให้ดูเป็นโหมดทบทวน
- ปรับปุ่มและ completion state ให้สื่อการเรียนรู้ต่อเนื่องมากกว่าการทำข้อสอบจบ

ผลที่คาดหวัง:

- เด็กยังตอบคำถามได้เหมือนเดิม
- ความรู้สึกทางอารมณ์เปลี่ยนจาก "สอบ" เป็น "ลองคิดและเรียนรู้"

### 5. Question Feedback

เป้าหมายคือเปลี่ยน feedback จากโทนตัดสิน เป็นโทนอธิบาย

สิ่งที่จะทำ:

- เปลี่ยนคำว่า "ถูกต้อง" และ "คำตอบไม่ถูกต้อง" ให้เป็นคำอธิบายที่เบากว่า
- ลดการใช้สีแดงที่เด่นเกินไป
- ทำให้ explanation card เน้นการเรียนรู้ ไม่ใช่การตัดสิน
- ลดความวูบวาบของตัวเลือกหากทำได้โดยไม่รื้อ component

ผลที่คาดหวัง:

- ผู้ใช้ไม่รู้สึกเสียหน้าหรือเครียดจากการตอบผิด

### 6. Random Fact

เป้าหมายคือคงแนว random fact ไว้ แต่ลดความรู้สึกเล่นเกินไป

สิ่งที่จะทำ:

- ปรับ wording ของปุ่มและข้อความให้เป็นมิตรขึ้น
- ลดความเด่นของ autoplay และ feedback เชิงเล่นสนุก
- ทำ action row ให้ดูสะอาดขึ้น

ผลที่คาดหวัง:

- ยังรู้สึกสดใหม่ แต่ไม่หลุดโทนแอปเรียนรู้

## Content Addition

บริเวณข้อความ:

- "พลังของนักเรียนที่ช่วยเป็นเกราะป้องกันภัยยาเสพติด"

จะเพิ่มคำอธิบายต่อจาก tagline หลักในหน้าแรก เพื่อช่วยขยายความว่าแอปนี้ไม่ได้คาดหวังให้เด็ก "เก่ง" หรือ "ผ่านการประเมิน" แต่ช่วยให้ค่อยๆ เรียนรู้, สังเกต, และปกป้องตนเองกับคนรอบตัวได้ดีขึ้น

แนวทางถ้อยคำ:

- สั้น
- อบอุ่น
- ไม่เหมือนสโลแกนราชการ
- ไม่เหมือนคำขวัญกิจกรรมโรงเรียน

## Testing

จะตรวจสอบอย่างน้อยดังนี้:

- แอป build ผ่านในระดับ analyzer ของไฟล์ที่แตะ
- ไม่มี linter error ใหม่ในไฟล์ที่แก้
- หน้า Home, Explore, Law, Quiz, Random Fact ยังเปิดได้ตามเดิม
- flow ของ quiz ยังทำงานเหมือนเดิม แม้จะเปลี่ยนข้อความและ visual emphasis

## Risks

- ถ้า soft wording มากเกินไป อาจทำให้เนื้อหาบางส่วนดูเบาเกินเจตนา
- ถ้าแตะ animation มากเกินไป อาจกระทบ feel เดิมของแอป
- ถ้าแตะหลายไฟล์พร้อมกันเกินจำเป็น อาจหลุดเป้าหมาย "ปรับเบาๆ"

## Recommendation

ให้ทำรอบนี้เป็น polishing pass เท่านั้น:

- เปลี่ยนเฉพาะถ้อยคำ
- ปรับเฉพาะ visual emphasis
- ลดเฉพาะจุดที่กดดันหรือแข็งเกินไป

ไม่ขยาย scope ไปถึงการรื้อหน้าหรือรีแฟกเตอร์ไฟล์ใหญ่
